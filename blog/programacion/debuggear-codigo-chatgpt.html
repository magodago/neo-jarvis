Debuggear es probablemente donde la IA mas impacto tiene para un programador. Encontrar un bug puede llevarte horas. Con el prompt adecuado, la IA lo encuentra en segundos y ademas te explica por que ocurrio.

Estos 2 prompts cubren el 90% de los escenarios de debugging que te encontraras.

Prompt 1: Debugging general

Cuando tu codigo no funciona y no sabes por que, este prompt aplica un enfoque estructurado.

Prompt
Eres un debugger experto. Este codigo no funciona: [PEGA CODIGO]. Comportamiento esperado: [describe]. Error: [PEGA ERROR]. Dame: 1) Causa raiz del problema, 2) Linea exacta donde falla, 3) Solucion con codigo corregido, 4) Explicacion de por que ocurria el error, 5) Test para verificar que la solucion funciona.

Prompt 2: Debugging de rendimiento

Cuando el codigo funciona pero es lento, este prompt identifica los cuellos de botella.

Prompt
Eres un optimizador de rendimiento. Este codigo es lento: [PEGA CODIGO]. Tiempo actual de ejecucion: [X]s para [Y] datos. Dame: 1) Operaciones con complejidad O(n^2) o mayor, 2) Cuellos de botella identifica-dos, 3) Codigo optimizado con explicacion de cada cambio, 4) Estimacion de mejora de rendimiento, 5) Alternativas de algoritmo si aplica.
